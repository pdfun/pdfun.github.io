Unity3d.Cocos2dx.Flask游戏接入iOS SDK

各种不同引擎的游戏接入iOS SDK大致的解决方案.

Unity3d

  • Unity能直接调用C的方法。首先把SDK相关的方法转换成C方法。
#if defined (__cplusplus)
extern "C"
{
#endif

    void sdkInit()
    {
        //这里直接调用SDK的相关方法进行初始化
    }

#if defined (__cplusplus)
}
#endif

转换后的文件(.h/.mm)可以通过Unity插件的方式导入，也可以在Unity导出的Xcode后再加到工程文件。

  • SDK回调处理。如果熟悉OC语法的话直接处理，在对应的回调里进行相关操作就行了。不熟悉可以在接收到回调的时候用UnitySendMessage向Unity发送通知消息,Unity接收到消息后再进行相应的操作。

Cocos2dx

  • 加入SDK相关文件后。可以在SDK相关的回调里直接调用C++的方法。 导入 cocos2dx头文件，使用CCNotificationCenter与Scene交互

  • SDK发送通知

#include "cocos2d.h"

const char * ccParamToken = [paramToken UTF8String];
    cocos2d::CCString *str = cocos2d::CCString::create(ccParamToken);
    cocos2d::CCNotificationCenter::sharedNotificationCenter()->postNotification("TOKEN", dynamic_cast<cocos2d::CCObject *>(str));
  • Cocos2dx接收通知
    CCNotificationCenter::sharedNotificationCenter()->addObserver(this, callfuncO_selector(HelloWorld::sdkNotification),"TOKEN", NULL);

Cocos2dx-lua

有这两篇文章足够了。

http://www.cnblogs.com/flyFreeZn/p/4152881.html

这篇文件介绍了下OC与lua如何实现交互，涉及Lua,C++,Objective-C三种语言。Lua调用Lua Binding的C++接口，C++接口调用混编的Objective-C接口，而Objective-C通过block形式的回调，将结果通知给C++，C++通过Lua的C API将最终结果返回给Lua。。。。。
